## [2.5.0] - 2026-06-15

### 变更类型：新增 + 重构 + 修改

### 产品定位调整
- **场景扩展**：从"深夜陪伴"扩展为"任何想倾诉的时刻"
- **一句话描述**：从"深夜陪伴式AI日记"改为"有温度的AI日记"
- **人格名称**：从"深夜陪伴"改为"朋友"

### 架构设计
- **四层模型**：明确系统架构分为四层
  - Persona Layer（AI人设）
  - Companion Layer（陪伴策略）
  - Memory Engine（记忆系统）
  - Data Layer（数据库）
- **核心原则**：记忆引擎是共享的，人设只影响表达方式
- **Memory Engine V2 架构**：新增 Temporal Reasoning 和 Thread Graph

### 新增功能
- **Temporal Memory（时间记忆）**：
  - 记忆 = 事件 + 时间 + 状态
  - 新增 temporal 字段：{startDate, endDate, isRecurring, periodLabel}
  - 时间推理：区分 2025年考试 vs 2026年考试
  - 纪念日触发：基于 temporal.startDate 自然提及
- **Memory Decay（记忆衰减）**：
  - 新增 decayScore 字段，每30天未引用 -0.1
  - decayScore < 0.3 自动归档
  - 被引用时 decayScore +0.2
- **Person 状态化**：
  - 闹翻了 ≠ 不存在
  - 关系变化更新 currentStatus，不删除记忆
- **Thread paused 状态**：
  - active → ongoing → paused → resolved

### 产品文档更新
- **功能2（记忆系统）**：新增时间记忆系统
- **数据库结构**：新增 temporal、decayScore、status 字段说明
- **全局设计原则**：新增四层架构说明

### 核心设计决策
- **记忆 = 事件 + 时间 + 状态**：不只是记住内容，还要记住时间
- **时间推理优先**：检索时优先匹配当前时间相关的记忆
- **纪念日触发**：基于真实时间的自然引用，不假装记得
- **人格可插拔**：新增人格只需换 System Prompt，不需要重做记忆系统

### 影响范围
- **AI服务**：时间推理能力加入记忆系统
- **未来功能**：人生线程视图、纪念日触发、AI成长叙事
- **人格系统**：多人格共享记忆，切换不失忆

### 兼容性说明
- 向后兼容：新增字段不影响已有功能

## [2.3.0] - 2026-06-15

### 变更类型：新增

### 新增功能
- **Memory Extraction Prompt**：从对话中提取记忆（每次0-3条）
  - 位置：`src/prompts/memory-extraction.md`
  - 包含记忆宪法四原则的具体实现
  - Importance 评分规则化：类型基础分 + 调整规则
- **Memory Consolidation Prompt**：合并/更新已有记忆
  - 位置：`src/prompts/memory-consolidation.md`
  - Thread 状态：active → ongoing → paused → resolved
  - Person 不删除，改状态（闹翻了≠不存在）
- **Memory Recall Prompt**：建议引用哪些记忆
  - 位置：`src/prompts/memory-recall.md`
  - 三种召回类型：direct/indirect/emotional
  - 只建议，不决定最终回复
- **Companion Response Prompt**：陪伴人格生成回复
  - 位置：`src/prompts/companion-response.md`
  - 决定是否引用记忆、如何引用、回复风格
  - 铁律：像朋友聊天，绝不展示"记忆行为"

### 代码更新
- **storage.js**：新增记忆相关 API
  - `referenceMemory(id)`：更新引用计数
  - `saveMemories(memories)`：批量保存
  - `executeConsolidation(actions)`：执行合并操作

### 数据库调整
- **memories表**：
  - 新增 `decayScore` 字段和索引（记忆衰减）
  - 新增 `status` 字段和索引（active/archived）

### 产品文档更新
- **功能2（记忆系统）**：新增第四个 Prompt 和记忆衰减系统
- **数据库结构**：新增 decayScore、status 字段说明
- **Thread 状态**：新增 paused 状态

### 核心设计决策
- **四层架构**：Extraction → Consolidation → Recall → Companion Response
- **Recall 只建议**：最终是否引用由 Companion 人格决定
- **记忆衰减**：decayScore 随时间自然衰减，避免记忆爆炸
- **Person 状态化**：关系变化更新状态，不删除记忆

### 文件结构
```
quiet-space/src/prompts/
├── memory-extraction.md    # 记忆提取
├── memory-consolidation.md # 记忆合并
├── memory-recall.md        # 记忆召回（建议）
└── companion-response.md   # 陪伴回复（决策）
```

### 影响范围
- **AI服务**：四个核心Prompt已设计完成，待代码实现
- **对话流程**：完整链路确定
- **产品差异化**：记忆系统 + 陪伴人格成为核心壁垒

### 兼容性说明
- 向后兼容：新增API和字段不影响已有功能

## [1.4.0] - 2026-05-28

### 变更类型：修改 + 新增

### 新增功能
- **全局设计原则**：粒子系统是核心视觉层、页面过渡必须有视觉过渡、氛围优先于功能
- **3D粒子时间线展廊**：三个入口（回忆/倾诉/碎碎念）共用3D粒子时间线展廊作为浏览界面，基于Three.js粒子系统，每个条目以粒子组成的图片形式展示

### 修改功能
- **倾诉（原聊天转日记）**：重新定义交互流程。进入后展示3D粒子时间线展廊浏览历史对话；新对话支持上传图片作为开题（图片转化为粒子背景）；预设背景待定；聊天界面由用户开启话题
- **回忆（原日记编辑与浏览）**：重新定义为时间线视图。默认展示3D粒子时间线，每篇日记显示标题、情绪标签、日期、内容预览；点击弹出预览卡片再进入编辑；筛选UI待定（需符合氛围设计）
- **碎碎念**：时间线展廊与回忆、倾诉共用视觉语言，展示碎片条目

### 功能调整
- **碎碎念**优先级：中优先级（MVP先做2D版本，后续升级Three.js 3D时间线）
- **预设背景**：标记为待定，后续补充具体预设内容

### 影响范围
- **首页入口**：三个入口进入后统一使用3D粒子时间线展廊
- **所有页面切换**：需添加视觉过渡动画，禁止硬切
- **环境场景**：粒子系统成为核心视觉层，固定场景（壁炉/夜雨）可与粒子系统结合

### 兼容性说明
- 向后兼容：功能定义更新不影响已有数据结构

## [1.0.0] - 2026-05-26

### 变更类型：新增

### 新增功能
- **聊天转日记**：用户和AI对话，AI自动生成日记草稿，用户可编辑保存
- **基础记忆系统**：三层记忆架构（短期对话/事件记忆/关系记忆），AI长期记住用户的故事
- **情绪记录**：后台自动分析对话情绪，生成情绪标签、强度和心理活动描述
- **日记编辑与浏览**：富文本编辑器 + 日历/列表/筛选三种浏览方式 + 全文搜索
- **AI人格系统**：预设人格选择 + 自定义参数滑块 + 可选的AI自适应成长
- **环境场景**：壁炉夜谈、夜雨窗边两个沉浸式环境主题（视觉+微动态+环境音）
- **API配置**：引导式三步配置（选提供商→填Key→测试连接），支持多模型切换

### AI增强调整
- 对话→日记生成：核心AI能力
- 情绪自动分析：后台静默分析
- 长期记忆提取：自动构建用户记忆库
- 日记润色：保留情绪不变的前提下优化表达

### 非功能性需求调整
- 性能：启动3秒内，动画60fps，内存不超过200MB，安装包不超过50MB
- 安全：API Key加密存储，纯本地数据，不收集用户数据
- 兼容性：优先Windows 10/11

### 技术栈调整
- 框架：Tauri 2（参考cc-switch）
- 前端：React/Svelte + Tailwind CSS + Canvas动画
- 数据库：SQLite（本地单文件）
- AI协议：OpenAI兼容格式

### 影响范围
- 这是初始版本，无已有功能受影响

### 兼容性说明
- 初始版本，无向后兼容性问题

## [1.1.0] - 2026-05-27

### 变更类型：修改 + 新增

### 新增功能
- **照片粒子背景**：用户上传照片，系统将照片转化为粒子效果作为氛围背景。粒子用照片颜色和位置还原照片内容，缓慢漂浮/呼吸
- **AI看图对话**：AI识别照片内容，以照片为引子自然开启对话。需多模态API支持（GPT-4o/Claude/Gemini）

### 修改功能
- **环境场景**：从"壁炉夜谈+夜雨窗边固定场景"改为"照片粒子背景为主，固定场景降级为后续版本备选"
- **API配置**：新增多模态API支持要求，纯文本模型自动降级

### 功能调整
- **照片粒子背景**优先级调整：中 → 高（MVP核心体验）
- **AI看图对话**优先级调整：新增即为高
- **环境场景（固定）**优先级调整：中 → 低（后续版本）
- **环境音**优先级调整：中 → 低（后续版本）

### 技术栈调整
- **粒子系统**：新增 Canvas 2D API（照片粒子背景渲染）
- **AI服务**：新增多模态API要求（图像输入支持）

### 影响范围
- 环境系统：从固定场景改为照片驱动，架构变化较大
- AI服务：需要支持多模态（图像输入），API配置流程需适配
- 前端：新增Canvas粒子渲染模块
- MVP范围扩大：新增2个高优先级功能

### 兼容性说明
- 向后兼容，v1.0.0的功能定义不受影响
- 固定场景从MVP移至后续版本，不影响已定义的核心功能

## 版本号规范

遵循语义化版本规范：`MAJOR.MINOR.PATCH`

- **MAJOR（主版本号）**：不兼容的 API 修改或重大功能变更
- **MINOR（次版本号）**：向下兼容的功能性新增
- **PATCH（修订号）**：向下兼容的问题修正

## 变更类型说明

### 新增（Added）
- 新增功能、新增 AI 能力、新增技术栈组件

### 修改（Changed）
- 修改现有功能、调整业务规则、优化用户体验

### 删除（Removed）
- 删除废弃功能、移除 AI 能力、移除技术栈组件

### 重构（Refactored）
- 代码重构、架构调整、性能优化
