# 技术术语表

日常开发中遇到的技术名词，用大白话解释。持续更新中。

---

## 一、视觉类型（Visual Types）

| 术语 | 大白话 |
|------|--------|
| **Dark Mode（深色模式）** | 整个界面底色是黑/深灰，不是白底。长时间使用不刺眼。 |
| **Glassmorphism（毛玻璃效果）** | 像透过磨砂玻璃看东西，半透明+模糊背景。 |
| **Neumorphism（新拟态）** | 按钮像从背景里"浮起来"或"压下去"，有轻微阴影和高光，很柔和。 |
| **Minimalism（极简主义）** | 界面上元素很少，大量留白，没有多余装饰。 |
| **Cinematic（电影感）** | 大面积暗色+低饱和度+光影氛围，像电影画面而不是普通App。 |
| **Gradient（渐变）** | 颜色从一种慢慢过渡到另一种，比如从深蓝渐变到黑色。 |
| **Blur（模糊）** | 背景或某些元素变模糊，像相机没对焦。用来制造景深感。 |
| **Opacity（透明度）** | 元素有多"透"。0%完全看不见，100%完全不透。 |
| **Saturation（饱和度）** | 颜色有多"鲜艳"。低饱和=颜色偏灰偏淡，高级感。高饱和=颜色很亮很刺眼。 |
| **Typography（字体排版）** | 用什么字体、多大、多粗、间距多少。直接影响阅读感受。 |
| **CSP（Content Security Policy）** | 网页的安全策略，限制哪些资源能加载。Tauri里配置在tauri.conf.json中。 |

## 二、网址/应用概念（Web/App Concepts）

| 术语 | 大白话 |
|------|--------|
| **URL（网址）** | 浏览器地址栏里的那串文字，比如 `https://example.com`。 |
| **SPA（单页应用）** | 整个App只有一个页面，内容动态切换，不会刷新整个页面。体验更流畅。 |
| **Routing（路由）** | App里不同"页面"之间的跳转规则。比如从首页跳到日记页。 |
| **API（接口）** | 两个程序之间传数据的"管道"。你的App调用大模型API，就是通过这个管道把用户说的话发给AI，再拿回来。 |
| **Local Storage（本地存储）** | 数据存在用户自己电脑上，不上传云端。 |
| **Database（数据库）** | 存放日记、聊天记录、记忆的地方。可以是本地文件，也可以是服务器上的。 |
| **Endpoint（端点）** | API的具体地址，比如"生成日记"是一个端点，"分析情绪"是另一个端点。 |
| **WebView** | 操作系统内置的浏览器内核。Tauri用WebView来显示界面，不需要打包整个浏览器。 |
| **Bundler（打包工具）** | 把你的代码、依赖、资源打包成一个可分发文件（exe、安装包等）的工具。 |

## 三、交互效果（Interaction Effects）

| 术语 | 大白话 |
|------|--------|
| **Hover（悬停）** | 鼠标移到某个元素上时，它发生变化（变色、放大、浮起）。 |
| **Transition（过渡）** | 变化不是瞬间完成的，而是平滑过渡。比如按钮颜色慢慢变。 |
| **Animation（动画）** | 元素动起来。比如火光闪烁、雨滴下落。 |
| **Micro-interaction（微交互）** | 小的、细微的反馈动作。比如打字时文字慢慢出现，发送按钮轻轻震动。 |
| **Scroll（滚动）** | 页面上下滑动。 |
| **Infinite Scroll（无限滚动）** | 划到底部自动加载更多内容，不需要点"下一页"。 |
| **Modal / Dialog（弹窗）** | 突然跳出来的窗口，覆盖在当前页面上面。比如设置面板、确认删除。 |
| **Tooltip（提示气泡）** | 鼠标悬停时出现的小提示文字。 |
| **Toast（轻提示）** | 屏幕上短暂出现的消息，比如"已保存"，几秒后自动消失。 |
| **Skeleton Screen（骨架屏）** | 内容还没加载出来时，先显示灰色占位轮廓，比转圈圈体验好。 |
| **Responsive（响应式）** | 界面能根据窗口大小自动调整布局。窗口缩小，内容重新排版。 |

## 四、动态技术（Animation/Dynamic Tech）

| 术语 | 大白话 |
|------|--------|
| **CSS Animation（CSS动画）** | 用样式代码控制元素动画，比如火光呼吸效果。性能好，适合简单动画。 |
| **Canvas（画布）** | 一个可以自由绘制图形、粒子的区域。雨滴、星空、粒子效果通常用这个。 |
| **WebGL / Three.js** | 在网页里做3D效果的技术。Three.js 是 WebGL 的封装库，原生 WebGL 代码极其复杂。详见第八章。 |
| **Lottie Animation** | 设计师做好动画，导出成JSON文件，代码直接播放。适合复杂的环境动画。 |
| **Particle System（粒子系统）** | 大量小点按照规则运动，模拟雨、雪、灰尘、火星飞溅。本项目的核心：图片像素 → 粒子位置/颜色 → Shader 控制运动。 |
| **Parallax（视差滚动）** | 背景和前景滚动速度不同，制造纵深感。 |
| **Scroll-triggered Animation（滚动触发动画）** | 页面划到某个位置时，动画才开始播放。 |
| **Frame Animation（逐帧动画）** | 一帧一帧播放图片，像翻连环画。火光如果要很真实可以用这个。 |
| **SVG Animation（SVG动画）** | 矢量图形的动画，放大不失真。适合图标、线条类的微动态。 |
| **HMR（Hot Module Replacement）** | 热更新。改了代码，页面不用刷新就自动更新。开发体验的核心。 |

## 五、可调节参数（Adjustable Parameters）

| 术语 | 大白话 |
|------|--------|
| **Slider（滑块）** | 一条横线，拖动小圆点来调节数值。比如"主动性：■■□□□"就是滑块。 |
| **Toggle（开关）** | 一个拨动开关，开/关。比如"是否开启AI自适应人格"。 |
| **Dropdown（下拉菜单）** | 点一下展开选项列表，选一个。比如选择AI人格类型。 |
| **Color Picker（颜色选择器）** | 让用户选颜色的工具。 |
| **Range（范围值）** | 一个最小值到最大值的范围。比如"AI回复长度：短→长"。 |
| **Config / Configuration（配置）** | 用户可以修改的各种设置项的总称。 |
| **Theme（主题）** | 整套颜色、字体、间距的预设方案。比如"夜雨窗边"是一套主题。 |
| **Preset（预设）** | 提前设好的一套参数组合，用户选一个就行，不用每个都调。 |
| **Default（默认值）** | 用户没改之前的初始设置。 |

## 六、开发工具与框架（Dev Tools & Frameworks）

| 术语 | 大白话 |
|------|--------|
| **Tauri** | 用Rust做底层+Web技术做界面的桌面应用框架。比Electron小很多。 |
| **Electron** | 把整个Chrome浏览器塞进去+网页做界面，打包成桌面应用。VS Code、Discord用的就是这个。 |
| **Vite** | 前端开发服务器和打包工具。改代码秒生效，不用等编译。 |
| **React** | Facebook做的前端UI框架。用组件的方式构建界面。 |
| **TypeScript** | 给JavaScript加了类型检查的增强版。写错了编辑器会提醒你。 |
| **Tailwind CSS** | 用短类名直接写样式的方式。不用来回切换CSS文件。 |
| **SQLite** | 轻量级本地数据库，整个数据库是一个文件。零配置，单文件备份。 |
| **Rust** | 一种系统编程语言，性能极好，内存安全。Tauri的底层用它写的。 |
| **Cargo** | Rust的包管理器和构建工具。类似前端的npm。 |
| **Node.js** | 让JavaScript能在浏览器之外运行的环境。前端开发必备。 |
| **npm** | Node.js的包管理器。用来安装各种前端工具和库。 |
| **WiX** | Windows安装包制作工具。Tauri用它来生成.msi安装程序。 |
| **MSVC** | 微软的C/C++编译器。Windows上编译Rust需要它。 |
| **rustup** | Rust的版本管理工具。用来安装、更新、切换Rust版本。 |

## 七、AI 相关（AI Concepts）

| 术语 | 大白话 |
|------|--------|
| **LLM（大语言模型）** | 超大规模的AI文本生成模型，比如GPT、Claude、DeepSeek。能理解和生成人类语言。 |
| **Prompt（提示词）** | 你给AI的指令/问题。写得好坏直接影响AI回复质量。 |
| **System Prompt（系统提示词）** | 给AI设定身份和规则的指令。用户看不到，但AI会遵守。比如"你是一个温柔的陪伴者"。 |
| **RAG（检索增强生成）** | 先从记忆库里找到相关资料，再让AI基于这些资料生成回复。让AI"记得"以前的事。 |
| **TTS（Text-to-Speech）** | 文字转语音。让AI的文字回复变成声音说出来。 |
| **STT（Speech-to-Text）** | 语音转文字。用户说话，AI转成文字来理解。 |
| **Token** | AI处理文本的最小单位。中文大约1个字=1-2个Token。API按Token数量收费。 |
| **Context Window（上下文窗口）** | AI一次能"看到"多少文字。超出的部分AI就"忘了"。 |
| **Embedding（向量化）** | 把文字转换成数学向量，方便AI搜索和比较相似内容。记忆系统的核心技术。 |

## 八、3D / WebGL 专项（3D & WebGL）

| 术语 | 大白话 |
|------|--------|
| **WebGL** | 浏览器里的 3D 绘图标准。让 JavaScript 能直接调用显卡画 3D 图形，不需要插件。 |
| **Three.js** | WebGL 的封装库。原生 WebGL 代码极其复杂，Three.js 把它变成好用的 API（Scene、Camera、Mesh 等）。 |
| **Scene（场景）** | 3D 世界里的"容器"，所有物体（粒子、模型、灯光）都放在 Scene 里。 |
| **Camera（相机）** | 决定从哪个角度观察 Scene。`PerspectiveCamera` 模拟人眼透视（近大远小），`OrthographicCamera` 是正交（没有透视变形）。 |
| **Renderer（渲染器）** | 每帧把 Scene + Camera 的结果画到 Canvas 上。`WebGLRenderer` 是最常用的。 |
| **BufferGeometry** | Three.js 里存储顶点数据的方式。粒子系统用它存位置、颜色、大小等属性，全部是 Float32Array，GPU 直接读取。 |
| **Points** | Three.js 专门画大量粒子的对象。和 Mesh（网格）不同，Points 只画点，不开销三角面，几万个也不卡。 |
| **Shader（着色器）** | 跑在 GPU 上的小程序，决定每个顶点/像素怎么画。分 Vertex Shader（顶点在哪）和 Fragment Shader（像素什么颜色）。 |
| **Vertex Shader（顶点着色器）** | GPU 上跑的第一步：决定每个点在屏幕上的位置。粒子的运动逻辑全写在这里。 |
| **Fragment Shader（片段着色器）** | GPU 上跑的第二步：决定每个像素的颜色。粒子的形状（圆形光点）、透明度在这里算。 |
| **GLSL** | 写 Shader 的语言，语法像 C，但跑在 GPU 上。不是 JavaScript。 |
| **Uniform** | Shader 的外部输入参数（如时间 uTime、扩散值 uDispersion）。JavaScript 传进去，Shader 里每帧都能读。 |
| **Attribute** | 每个顶点独有的数据（如位置、颜色）。存 BufferGeometry 里，GPU 并行读取。 |
| **Simplex Noise（单纯形噪声）** | 一种生成自然随机运动的算法。比 Perlin Noise 更快更均匀。用来让粒子运动看起来"有机"而不是机械。 |
| **Additive Blending（加法混合）** | 粒子叠加时颜色相加（不是覆盖）。越叠越亮，产生发光/光晕效果。粒子系统的标配。 |
| **BufferAttribute** | Three.js 里把 JavaScript 数组传给 GPU 的方式。比如把粒子位置数组变成 GPU 能读的 attribute。 |
| **Pixel Ratio（像素比）** | 设备的物理像素/逻辑像素比。Retina 屏是 2，普通屏是 1。不设置的话 Retina 屏上的 Canvas 会模糊。 |

## 九、图像处理（Image Processing）

| 术语 | 大白话 |
|------|--------|
| **getImageData()** | Canvas API，读取图片的原始像素数据（RGBA 四个通道，0~255）。图片转粒子的核心 API。 |
| **Alpha Channel（Alpha 通道）** | 像素的透明度。0=全透明，255=不透明。处理图片时只取 alpha > 某个值的像素来生成粒子。 |
| **Pixel Mapping（像素映射）** | 把图片的 2D 像素坐标 (x,y) 转换成 3D 空间坐标 (x,y,z) 的过程。z 轴通常由亮度决定。 |
| **Image Sampling（图像采样）** | 读取图片像素信息的过程。缩放图片后逐像素扫描，就是采样。 |

## 十、手势/体感控制（Gesture Control）

| 术语 | 大白话 |
|------|--------|
| **MediaPipe Hands** | Google 开源的手势识别库。摄像头拍手，它能识别 21 个手部关键点的坐标。 |
| **Hand Landmarks（手部关键点）** | MediaPipe 识别出的 21 个点（指尖、关节等）。用这些点的坐标距离可以判断手势（捏合、张开等）。 |
| **Pinch（捏合）** | 拇指和食指指尖靠近的手势。在粒子系统里控制粒子聚拢。 |
| **Camera Utils** | MediaPipe 的摄像头工具库，封装了 getUserMedia + 逐帧发送给 Hands 处理的逻辑。 |
